Extended Debug for the Emulator

F11 or from the view menu.

  • c - compare several bytes.
    
v1 db  1,4,3,4,5,6,7
v2 db  1,5,3,0,5,6,7
    To compare v1 and v2 type the following:

    c v1 L 7 v2

    L is the size of the array. Without the L parameter 8 bytes are compared.

    It is also possible to use registers as parameters:

    c SI L 20h DI

    The command outputs only those addresses with values that are different.

  • i - input and display a byte from a specified port:

    i 123

    iw - input and display a word.

  • o - output byte to a specified port:

    o 123 55

    ow - output a word.

  • d - dump 128 bytes at specified address, for example:

    d 100

    If no segment is specified CS is used by default. Square brackets are optional.
    d command without parameters shows next 128 bytes.

    L parameter can be used to specify the number of bytes to dump. Dump 10 bytes starting from an offset of a variable called var:

    d var L A

    If there's a variable L, declared as L db 5, brackets must be used:

    d [L]
    d [L] L 1

  • t - single step.

  • p - step over (procedure or interrupt).

  • r - show all registers and current instruction at CS:IP.

    • rf - view/set flags.
    • r [register name] - view/set register value.




Note: All output values are in HEX. All input parameters are be in HEX. "h" suffix is not required.
